Our Idea

  • This application is committed to improving the quality of life of people at all levels of the medical department.
  • This application will store the necessary information for everyone. In addition, will supply / invoice if required.

Motivation

  • Human society is constantly working tirelessly for the progress of human civilization and the improvement of human life.
  • Provide accurate information on when the incidence of a disease is increasing and decreasing. It will give accurate information about the percentage of any disease.

Requirements

Requirement is something that is needed or that must be done. As we are deploying our own web based software, the requirements are given below are from our consideration and if our customer need more we will obviously provide them after the next maintenance.

Raw requirements:

  • This must be a web based system
  • There will some introductory pages for non-registered users.
    • These pages will contain home, about, services, advice from admin and contact pages.
  • There will be a log in option for registered users and a sign in option for non-registered users if they are interested with this system.
  • User can follow any doctor if they want
  • User may have a post option, which will help them to express their sickness among the followed doctors and their friends whom they followed.
  • User can store their all prescription and drug list and even their lab report also.
  • Patient can message if he/she has an appointment of that doctor.
  • Doctor can see patient’s prescription, drug list and lab report too.
  • A doctor can have both doctor id and a patient id.

Functional requirements:

  1. Web based system.
  2. Five web pages like Home, Services, Advice, About, Contact for a visitor.
  3. Log in.
  4. Sign in.
  5. Option for creating a post.
  6. Option to follow someone.
  7. Specific panel for prescription, drug history, lab report, .
  8. Messaging option between doctor and patient.
  9. Doctors are able to see the patient’s total history
  10. Web system should be responsive.
  11. User can send review via contact page.
  12. All info should be filled while login or sign up.

Non-Functional requirements:

  1. Security: The software must remain resilient in the face of attacks like DOS attack or SQLi and XSS/CSRF protected.
  2. Accessibility: People with the widest range of capabilities can use the system.
  3. Availability: The system is able to function during normal operating times.
  4. Confidentiality: The system protects sensitive data and allows only authorized access to the data.
  5. Efficiency: The data are maintained by the software system in terms of accuracy, authenticity and without corruption.
  6. Reliability: The software system consistently performs the specified functions without failure.
  7. Safety: The system prevents harm to people or damage to the environment.
  8. Survivability: The system continues to function and recovers in the presence of a system failure.
  9. Usability: The user is able to learn operate, prepare inputs and interpret outputs through interaction with a software system.
